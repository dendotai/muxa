# Installation

muxa can be installed globally using your preferred package manager.

## Package Managers

<Tabs defaultValue="bun">
  <Tab value="bun" label="Bun">
    ```bash
    bun install -g @den-ai/muxa
    ```
  </Tab>
  <Tab value="npm" label="npm">
    ```bash
    npm install -g @den-ai/muxa
    ```
  </Tab>
  <Tab value="pnpm" label="pnpm">
    ```bash
    pnpm add -g @den-ai/muxa
    ```
  </Tab>
  <Tab value="yarn" label="Yarn">
    ```bash
    yarn global add @den-ai/muxa
    ```
  </Tab>
</Tabs>

## Requirements

- **Node.js** >= 16.0.0
- **mprocs** (automatically installed as dependency)

## Verify Installation

After installation, verify that muxa is available:

```bash
muxa --version
```

You should see the version number displayed.

## Platform Support

muxa works on:
- ✅ macOS
- ✅ Linux
- ✅ Windows (with WSL)

<Callout type="info">
**Note for Windows users**: muxa relies on Unix-style shell commands. For the best experience on Windows, use WSL (Windows Subsystem for Linux).
</Callout>

## Troubleshooting

### Command not found

If you get a "command not found" error after installation, you may need to add the global bin directory to your PATH:

<Tabs defaultValue="npm">
  <Tab value="npm" label="npm">
    ```bash
    # Find npm global bin directory
    npm bin -g
    
    # Add to your shell profile (~/.bashrc, ~/.zshrc, etc.)
    export PATH="$PATH:$(npm bin -g)"
    ```
  </Tab>
  <Tab value="yarn" label="Yarn">
    ```bash
    # Find yarn global bin directory
    yarn global bin
    
    # Add to your shell profile
    export PATH="$PATH:$(yarn global bin)"
    ```
  </Tab>
  <Tab value="pnpm" label="pnpm">
    ```bash
    # Find pnpm global bin directory
    pnpm bin -g
    
    # Add to your shell profile
    export PATH="$PATH:$(pnpm bin -g)"
    ```
  </Tab>
</Tabs>

### Permission errors

If you encounter permission errors during global installation:

1. **Recommended**: Configure npm to use a different directory for global packages
2. **Alternative**: Use a Node version manager like nvm, fnm, or volta
3. **Not recommended**: Use sudo (can cause permission issues later)
